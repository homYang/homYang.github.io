【Algorithms】排序算法总结

2022-04-07 
各种排序算法，每次学的时候觉得这次肯定懂了，几个月后写代码却又阿巴阿巴。尝试自己总结一次，看看几个月后病情会不会好转（😊

以下排序算法总结的先后次序，根据使用（面试）频率从高到低排

快速排序（不稳定，nlogn，in-place）

1、基本思想：选取start作为基准pivot，设计 partition 函数：将数据分为【小于pivot，pivot，大于pivot】。递归调用 partition()，分别对基准两边进行快速排序。

2、复杂度：
· nlogn
· 最坏情况即数据有序的情况下，每次分区都只分出一个1和n-1，时间复杂度n^2。

3、partition() 具体实现：i从start+1出发，指向大于基准的数据位置；j从end出发，指向小于基准的数据位置。交换i和j指向的数据。重复以上动作，直到i>=j。最后 j 指向的是左边区域最后一个小于基准的数，所以交换 start 和 j 位置的数据，实现了左小右大的局面。start >= end 时不再进入 partition()。注意边界条件！！

4、优化：
· 排序前先shuffle待排序数组
· 哨兵。去掉边界条件的判断
· 小数组切换到插入排序
· 三取样切分

代码优化

代码j==lo的判断其实是冗余的，因为只有a[j]>a[lo]时，j才会递减。当j到达lo时必定停止移动。
若想去掉i==hi到判断，需要在shuffle后把数组最大值放到hi位置。i从lo出发时不必担心跑出hi，因为i只有比基准小才会递增，所以遇到hi时i必定停止移动。
子数组处理：右边部分的第一个值可以作为左边部分的hi

常见应用：find top k

归并排序（稳定，nlogn）

1、基本思想：如何排序一个数组，假设数组分为左右两个子数组，且子数组已排好序，那么有序合并两个子数组，任务就完成了。这个假设中，两个子数组如何有序的问题跟原始问题一致。因此可以递归解决子问题，即分别排序左右子数组，合并两个子数组。

void merge(const vector<int>& a, int lo, int mid, int hi) { // merge a[lo..mid] and a[mid+1..hi]
&emsp;&emsp;vector<int> aux(a);
&emsp;&emsp;int i = lo; int j = mid+1;
&emsp;&emsp;for(int k = 0; k <= hi; ++k) {
&emsp;&emsp;&emsp;&emsp;if(i > mid) aux[k] = a[j++];
&emsp;&emsp;&emsp;&emsp;else if(j > hi) aux[k] = a[i++];
&emsp;&emsp;&emsp;&emsp;else if(a[i] < a[j]) aux[k] = a[i++];
&emsp;&emsp;&emsp;&emsp;else aux[k] = a[j++];
&emsp;&emsp;}
}

2、递归（自顶向下）方法，空间复杂度除了辅助数组外，还有logn的调用栈深度

3、自底向上方法可以节约调用栈的空间消耗。每次两两归并，再四四归并，再八八归并，一直下去。

【Algorithms】我对动态规划的理解Ⅰ

2022-03-20

一、题型辨别

求最值（最大最小值/最长串/最短路径/。。），求可行解的总个数，0-1背包问题。

二、分析

以上题型都有几个共同点

1、大问题可分解为小问题，有最小可行解，存在重复子问题

【 如果第一次接触，觉得难以理解，先牢记典型大问题依赖子问题🌰：
斐波那契数列 f(n)=f(n-1)+f(n-2) ，解决了 上一个 和 上上个问题，就可以解决现在的问题。先不要细究如何分解子问题 】

2、小问题的每一步本质都是“选or不选”的问题

难点：动态规划类问题难在：

（1）找“大小问题之间的关系式”
（2）优化内存

易错点：处理边界问题

接下来我们从例题中感受，特别是“选or不选”的问题，为解题关键思想。而“优化内存”是提分所在。

三、第一类：单序列问题

https://leetcode.com/problems/min-cost-climbing-stairs/
剑指 Offer II 088. 爬楼梯的最少成本
数组的每个下标作为一个阶梯，第 i 个阶梯对应着一个非负数的体力花费值 cost[i]（下标从 0 开始）。
每当爬上一个阶梯都要花费对应的体力值，一旦支付了相应的体力值，就可以选择向上爬一个阶梯或者爬两个阶梯。
请找出达到楼层顶部的最低花费。在开始时，你可以选择从下标为 0 或 1 的元素作为初始阶梯。
示例：
输入：cost = [10, 15, 20]
输出：15
解释：最低花费是从 cost[1] 开始，然后走两步即可到阶梯顶，一共花费 15 。

分析确定状态转移方程

爬楼梯可以选择爬一阶或两阶，说明了当前位置 n 可能来自 n-1 或 n-2，是不是和斐波那契数列很像？

那么当前位置的最小花费，就是 n-1 和 n-2 中的最小花费，加上当前位置的花费。得到：
f(n) = min( f(n-1), f(n-2) ) + cost[n]

// Recursive Top Down - O(2^n) Time Limit Exceeded
int minCostClimbingStairs(vector<int>& cost) {
  int n = cost.size();
  return min(minCost(cost, n-1), minCost(cost, n-2));
}
int minCost(int[] cost, int n) {
  if (n < 0) return 0;
  if (n==0 || n==1) return cost[n];
  return cost[n] + min(minCost(cost, n-1), minCost(cost, n-2));
}

写出递推公式，再处理好边界问题（n>=2）其实答案就出来了。但这样的代码通常会超时，因为很多子问题被重复计算（时间效率极差）。看看下图就可以明白。因此，画递归树是非常重要的，许多题目可以利用递归树求解，后续会有实例。

使用缓存

为了避免重复计算，我们可以把求解过的结果保存起来。

// Top Down Memoization - O(n) 1ms
int minCostClimbingStairs(vector<int>& cost) {
  int n = cost.size();
  vector<int>dp(n);
  return min(minCost(cost, n-1), minCost(cost, n-2), dp);
}
int minCost(int[] cost, int n, vector<int>& dp) {
  if (n < 0) return 0;
  if (n==0 || n==1) return cost[n];
  if (dp[n] != 0) return dp[n];
  dp[n] = cost[n] + min(minCost(cost, n-1), minCost(cost, n-2));
  return dp[n];
}

自下而上解决问题，迭代方式

就是先计算子问题。
// Bottom up tabulation - O(n) 1ms
int minCostClimbingStairs(vector<int>& cost) {
  int n = cost.size();
  vector<int>dp(n);
  for (int i=0; i<n; ++i) {
    if (i<2) dp[i] = cost[i];
    else dp[i] = cost[i] + min(dp[i-1], dp[i-2]);
  }
  return min(dp[n-1], dp[n-2]);
}

空间复杂度为O(1)的迭代代码

其实我们只需要用到 f(n-1), f(n-2)，并不需要保存 n 个解集，优化后代码为:
int minCostClimbingStairs(vector<int>& cost) {
  int curCost = 0;
  int ppCost = 0;
  int pCost = cost[0];
  for(int i = 1; i < cost.size(); ++i) {
    curCost = min(pCost, ppCost) + cost[i];
    ppCost = pCost;
    pCost = curCost;
  }
  return min(ppCost, pCost);
}

动态规划优势在于，减少了重复计算，还能节约内存

Challenge. Would adding edge v–w to tree T create a cycle? If not, add it.
How difficult?
・V <– run DFS from v, check if w is reachable (T has at most V – 1 edges)

Challenge. Would adding edge v–w to tree T create a cycle? If not, add it.
Efficient solution. Use the union-find data structure.
・Maintain a set for each connected component in T.
・If v and w are in same set, then adding v–w would create a cycle.
・To add v–w to T, merge sets containing v and w.

Proposition. Kruskal's algorithm computes MST in time proportional to E log E (in the worst case).

Remark. If edges are already sorted, order of growth is E log* V.

Prim 算法

lazy implementation

  1. 从顶点0开始，标记顶点 “已加入”
  2. 把“已加入”顶点的所有边加入优先队列（最小堆）
  3. 把 minPQ 边加入 MST。若边的两端都 “已加入”，忽略这条边
  4. 否则，边的另一个顶点标记 “已加入”，goto第二步
  5. 重复，直到树中有 V-1 条边为止

算法复杂度

Proposition. Lazy Prim's algorithm computes the MST in time proportional to E log E and extra space proportional to E (in the worst case).

性质

方法：

2、边的松弛 Edge relaxation

最短路径的实现都基于一个称为松弛的简单操作。

放松边 e=v->w 意味着：检查 s到w的最短路径 是否 s经过v再到w，如果是，称e的放松是成功的。

松弛术语来自：用一根橡皮筋沿着连接两个顶点的路径紧紧展开的比喻：放松一条边类似将橡皮筋转移到一条更短的路径上，从而缓解了橡皮筋的压力。

3、最短路径算法理论基础 —— 最优性条件（Optimality condition)

4、通用算法

Efficient implementations. How to choose which edge to relax?
Ex 1. Dijkstra’s algorithm (nonnegative weights).
Ex 2. Topological sort algorithm (no directed cycles).
Ex 3. Bellman-Ford algorithm (no negative cycles).

Dijkstra 算法 （权重非负的加权无向图）

  1. 初始化distTo[] 为正无穷，起始顶点s distTo[s] = 0，放松与s连接的所有边，更新edge[] distTo[]
  2. 类似Prim算法，标记“已放松的顶点”，把它的边加入优先队列（最小堆）
  3. 放松 minPQ 的那条边。（因为这条边是连接s 的最短路径了，先放松它，它连接的顶点也会得到最短路径）
  4. 重复，直到所有顶点“已放松”


Proposition. Dijkstra’s algorithm computes a SPT in any edge-weighted digraph with nonnegative weights.
Pf.
・Each edge e = v→w is relaxed exactly once (when v is relaxed),leaving distTo[w] ≤ distTo[v] + e.weight().
・Inequality holds until algorithm terminates because:
– distTo[w] cannot increase <– distTo[] values are monotone decreasing
– distTo[v] will not change <– we choose lowest distTo[] value at each step (and edge weights are nonnegative)
・Thus, upon termination, shortest-paths optimality conditions hold.

Dijkstra’s algorithm seem familiar?
・Prim’s algorithm is essentially the same algorithm.
・Both are in a family of algorithms that compute a graph’s spanning tree.

Main distinction: Rule used to choose next vertex for the tree.
・Prim’s: Closest vertex to the tree (via an undirected edge).
・Dijkstra’s: Closest vertex to the source (via a directed path).

按照拓扑顺序放松顶点 （无环加权有向图 edge-weighted DAGs）

Proposition. Topological sort algorithm computes SPT in any edgeweighted DAG in time proportional to E + V.
Pf.
・Each edge e = v→w is relaxed exactly once (when v is relaxed),leaving distTo[w] ≤ distTo[v] + e.weight().
・Inequality holds until algorithm terminates because:
– distTo[w] cannot increase <– distTo[] values are monotone decreasing
– distTo[v] will not change <– because of topological order, no edge pointing to v will be relaxed after v is relaxed
・Thus, upon termination, shortest-paths optimality conditions hold

总结 Dijkstra算法 和 拓扑排序算法 可以解决单源最短路径的证明：
两个算法都满足了最短路径的最优性条件。他们用不同方式保证了：如果v是从起点可达的，所有 v->w 的边只会被放松一次，不等式 distTo[w] ≤ distTo[v] + e.weight() 成立。
保证了 distTo[v] 是不会变化的：
1. 权重非负的情况下，Dijkstra算法保证：从起始顶点的最小权重边开始放松，之后的放松操作都不可能使任何 distTo[]小于 distTo[v]。也就是 distTo[放松了的顶点] 不会再变化（已经是最小权重了嘛）；
2. 无环的时候，拓扑顺序保证：放松了的顶点，后面再没有顶点指向它，也就是放松的顶点不会再变化（都没机会改变了嘛）。
因此放松 v->w 的边时，要么放松无效（ 原有的 distTo[w] ≤ distTo[v] + e.weight() ）；
要么放松成功（ 原有的 distTo[w] > distTo[v] + e.weight()， 放松后 distTo[w] = distTo[v] + e.weight()）

【读书笔记】自控力--大脑的弥天大谎：为什么我们误把渴望当幸福？

2022-03-04

一、前情提要

根据过去的研究报告，小白鼠很讨厌电击。而1953年，两名年轻科学家（奥米兹和米尔纳）将一个电极植入小白鼠脑袋，这只小白鼠非常希望被电击。原来这两位科学家是社会心理学家，他们糟糕的实验技巧，阴差阳错发现了大脑中一个受刺激会产生强烈快感的区域。这一实验让我们看的自己渴望、被诱惑、上瘾的种种体验。我们还会发现，神经营销学如何利用这一原理为我们制作欲望，操控我们的大脑，以及我们如何才能抵抗这种欲望。

二、实验详情 – 奖励的承诺

（对实验不感兴趣可跳过这段）

奥米兹和米尔纳 想证明刺激的小白鼠区域会带来多大的快感，他们进行了以下实验。

  1. 小白鼠禁食24小时，放它在一个短管中间，管两端都有食物。通常小白鼠会跑到一头开始吃东西。但如果它在此之前受到电击，它便会待在原地，一动不动。和一份有保证的食物奖励比起来，它更愿意等待可能出现的另一次点击。
  2. 测试小白鼠是否主动寻求电击：有一根杆杠被按压时小白鼠会受到电击，小白鼠一旦发现杆杠的作用，就会每5秒钟电击自己一下。小白鼠显得毫不满足，它们一直按，直到力竭而亡。如果自我折磨能够刺激大脑，它就能忍受这种折磨。
  3. 奥尔兹把杆杠放着一张电网的两端，结果小白鼠很乐意在电网上跑来跑去，直到它们烧焦的爪子疼得无法继续跑。

没过多久，精神病学家觉得这个实验对人类来说也是可以尝试的。

  1. 罗伯特 希斯在病人大脑中植入电极，并给他们一个控制盒。结果他们表现得跟小鼠如出一辙，他们平均每分钟会电击自己40次。休息的时候，虽然病人们承认自己饿了，仍不愿意停下电击去吃东西。
  2. 实验人员提出终止实验或切断点击时，有一个病人提出强烈抗议。
  3. 另一个被试者在电流切断后仍按了200多下按钮，直到实验人员要求他停下来。

补充实验背景

当时占主导地位的科学范式是行为主义，认为唯一值得测量的东西就是行为。想法/感觉 都是浪费时间的东西。这可能就是为什么希斯早期的报告没有记录病人在自我刺激时有什么感觉。科学家推断被试者把精神的快感当作“奖励”。

触发进阶发现的实验

有一名病人有间歇性睡眠症，实验人员给他一个便携植入设备，帮助他保持清醒。这位病人描述自我刺激的感觉时说，那“令人非常沮丧”。虽然他“经常性地、有时近乎疯狂地按按钮”，但他一直没能达到实验中那种满足感。自我刺激令他焦虑不安，而不是辛福快乐。他看起来更像是患强迫症的人。

实验的猜测

小白鼠之所以刺激自己，是因为大脑告诉它们，只要再按一次杆杠，奇妙的事情就会发生。

人脑的“奖励”系统

奥米兹和米尔纳发现的不是快感中心，而是现在神经科学家称为“奖励”系统的东西。他们刺激的区域是人脑最原始的动力系统的一部分。这个系统逐步进化，驱使我们采取行动和消耗体能。每当这个区域收到刺激，大脑就会说：“再来一次！这会让你感觉良好！”但刺激本身并不会带来满足感。

并非只有大脑中的电极能激活这个系统。世界上充满着带来刺激的东西，从饭店菜单和直邮广告，到乐透彩票和电视广告，都能让我们变成真人版的奥米兹和米尔纳的小白鼠，去追寻对快乐的承诺。这时候，我们的大脑就会对“我想要”的东西深深着迷，而说“我不要”更加困难。

三、“我想要”的神经生物学原理

奖励系统是怎么迫使我们行动的？当大脑发现获得奖励的机会，就会释放多巴胺（一种神经递质），它告诉大脑的其它部分“怎样才能让贪婪的我们得手”。大量的多巴胺并不能产生快乐的感觉，而是一种激励。我们发现了如何才能得到快乐，而且愿意为了获得这种感觉付出努力。

2001年，斯坦福神经科学家布莱恩-克努森 证明了多巴胺会促使人们期待得到奖励，但不能感觉到获得奖励时的快乐。研究如下：

克努森扫描被试者的大脑，让他们看到某个符号时就期待自己能赢钱，想要赢球就按一个按钮。只要符号一出现，人类大脑中释放多巴胺的“奖励中心”就会发生反应，但当被试者真的赢了钱，这个区域反而安静下来。大脑中另一个区域产生了赢钱的快感。

多巴胺控制的是行动，不是快乐。

奖励系统活跃时，感受到的是期待，而不是快乐。

任何让我们高兴的东西都会刺激奖励系统，例如美食，商店门口的半价招牌，性感陌生人的微笑，承诺让你变得富有的商业广告。当多巴胺劫持了你的注意力，大脑只会想如何获得或重复那个触发它的东西。这是大自然的办法，它保证你不会因为不愿意采集浆果而被饿死，或是因为觉得很难吸引潜在伴侣而加速人类灭亡。进化根本不关心你快乐与否，但它会利用对快乐的承诺，让我们不停地为生计奔忙。大脑正是靠对快乐的承诺让你不断狩猎、采集野果、工作和求爱，而不是让你直接感受快乐。

你也可以思考一下色情图片对我们奖励系统的影响。我们发现互联网上关于色情的东西随处可见，而每一次性“机会”的本能，会使人们最终对色情网站上瘾，或是对某些涉性广告上瘾，比如除臭剂和高档牛仔裤的广告。

现代科技“及时行乐”的特点，加上原始的激励系统，让我们成了多巴胺的奴隶。

  • 从前：狂按电话答录机按钮、查收新消息的刺激感
  • 接着：电脑会告诉我们 “你收到了新邮件！”
  • 现在：Facebook、Twitter、电子邮件和短信息。因为我们知道自己可能会收到新消息，或者下一个更像的YouTube视频可能让我们捧腹大笑，我们就不停刷新，点击下一个链接，就像得了强迫症一样查看我们的设备。这些设备就这样俘获了我们，让我们不断要求更多。我们点击鼠标，就跟小白鼠想再来一次电击一样。我们追寻着难以捉摸的奖励，直到最终觉得满意。
  • 手机、互联网和其它社交媒体可能是无意激活了我们的奖励系统，但电脑和电子游戏的设计者是有意识地控制了人们的奖励系统。“升级” “获胜” 随时可能出现，游戏就这样激发人的兴趣，也是难以戒掉的原因。电子游戏刺激和使用苯丙胺时产生的多巴胺一样多，具有不确定性的“得分” “升级” 让你的多巴胺神经元不停燃烧。

2005年，28岁的韩国锅炉修理工在连续50小时奋战“星际争霸”后死于心血管衰竭。这件事很难不联系到奥米兹和米尔纳实验中力竭而亡的小白鼠。

深入剖析：是什么让你的多巴胺神经元不停燃烧？食物？酒精？购物？Facebook？还是其它？这一周试着观察是什么吸引了你的注意力。是什么给了你奖励的承诺，强迫你去寻求满足感？

四、帕金森症的处方

帕金森症的病因是脑细胞中缺少多巴胺，治疗方式是同时服用两种药物：左旋多巴和多巴胺受体激动剂。这在减轻帕金森主要症状的同时也带来了难以预料的新问题。

  • 一个54岁女人突然对曲奇饼干、意大利面产生强烈的欲望，她会很晚都不去睡觉，一直不停狂吃。
  • 一个52岁男人养成每天赌博的习惯，在赌场待了36小时，花光所有积蓄。
  • 一个49岁男人觉得非常痛苦，因为食量大增、总想喝酒、妻子说他“性欲过强”并打电话向警察求助。

这些例子有些极端，但当你被奖励的承诺吸引时，你大脑中的情况和这些人没有多大差别。当多巴胺给我们大脑安排寻找奖励的任务时，我们就展现了我们最敢于冒险、最冲动、最失控的一面。最重要的是，如果奖励迟迟没到，奖励的承诺（和一想到要停下来就不断增长的焦虑）足以让我们一直上瘾。如果你是人类，你会掏空钱包、填饱肚子——这还是好的。严重的话会发现自己患上了强迫症。

五、神经营销学的崛起

当奖励的承诺释放多巴胺时，你更容易受到其它形式的诱惑。比如色情图片使男性更容易在经济方面冒险，幻想中乐透彩票会让人饮食过量。这两种对无法得到的奖励的幻想会给你带来麻烦。大量分泌的多巴胺会放大“及时行乐”的快感，让你不再关注长期的后果。

发现这事的人，是想从你身上赚钱的人。

零食业的方方面面都设计得让我们更有购买欲。比如，大型食品公司中菜谱搭配适当的糖类、盐类和脂肪，让你的多巴胺神经元处于兴奋状态；乐透彩票广告鼓励你去想象中大奖后拿着100万美元去做些什么。杂货店老板也不傻，他们把最具诱惑力的商品放着前面和中间。

走进商店时，第一眼看到的就是面点区免费试吃的样品。样品包含了两个最大的奖励承诺——免费和食物。（如果发放样品的人很有魅力，那就是第三个承诺了。那你就完蛋了。）

品尝甜食样品的人更容易购买放纵自己的食物，比如牛排和蛋糕，或是打折商品。（还会让被试者更想获得与购物无关的奖励，包括去度假、看一场爱情电影、泡温泉。这说明了商家无论推销什么产品，都应该在买东西时拼命供应饼干。）

大脑的奖励系统对新鲜感和多样性也会有反应。所以说餐厅会不断在标准套餐的基础上推出新品，服装商会为压箱底的旧款式推出新颜色。

  • 我来一杯普通的咖啡吧？嗯，到店里我就点这个。噢，菜单上好像有新东西。你又激动了吧！

  • 你最喜欢的服装目录里有件绞花针织毛衣？太没劲了。但是，等一下，这款毛衣有焦糖棕色和奶油黄色的？今天你又多巴胺分泌旺盛了吧！

  • 还有价格的巧妙设置，能保证你大脑的原始部分想储存这些稀有资源。从能让你觉得“买到便宜货”的东西，“买一送一”的承诺到高喊“减价60%”的招牌。特别有效的方法上，在打折零售价旁边加一个高得离谱的“建议零售价”。你的大脑快速计算出省了多少钱，而且（毫无逻辑）认为这个差价上你赚到的钱。简直就是白送嘛！如果再给你时间限制或数量限制的话（“店内促销中午截止”、“特供最后一件”），你就会像在大草原抢夺最后一点食物那样，冲上去大抢特抢。

  • 卖家通过味觉让消费者产生不存在的欲望。让人食欲大开的气味可能很快就会引发你的奖励承诺。当你接触到气味分子，大脑就会开始寻找气味的来源。你要知道，香气不是店里食物飘出来的，而是精心设计的装置释放到人行道上。
  • 佛罗里达一家医院里，核磁共振部门的医生在等候区释放沙滩椰树和海洋气味，成功降低了最后一分钟取消检查预约的概率。

六、做个多巴胺侦探

当你开始关注的时候，你就会发现有多少陷阱中等着诱惑你、你的多巴胺神经元和你的钱。

作者的学生表示，这种观察让自己充满动力。他们中观察商家技巧时感到高兴。这也有助于发现一些购物的秘密，比如为什么有些东西在商店里看起来那么吸引人，但回到家却那么令人失望。一位女士终于弄明白了，为什么她无聊的时候总想冲进去食品商店，不是为了买食品，只是为了进去看看。

虽然我们所处的世界总让我们产生欲望，但只要我们用心观察，就能看透一些东西。知道那是怎么一回事并不能完全消除你的欲望，但能让你有机会抗争一下，锻炼一下“我不要”力量。

深入剖析：谁在控制你的多巴胺神经元？观察一下，零售商和营销人员如何刺激奖励承诺。把逛商店或看广告当作游戏。你闻到了什么？看到了什么？听到了什么？当你知道这些暗示都是经过精心设计，专门要诱惑你上钩时，你就能看清他们到底是什么，也能成功抵御了。

七、让多巴胺发挥作用

运用从神经营销学家那里学到的，试着把我们最不喜欢东西“多巴胺化”。在承诺奖励之后，让我们不开心的家庭琐事可能变得很有吸引力。当奖励在遥远的未来才能实现时，我们可以通过幻想最终的奖励（不是像彩票广告一样哦）从神经元里挤出一点多余的多巴胺。

戒酒和戒毒过程中，最有效的干预疗法“鱼缸法”：通过药物检测的病人有机会从鱼缸抽出一张纸。一半的纸写着一个价格从1～20美元不等，其中一张纸写着100美元。另一半写着“继续努力”。这应该不算什么激励作用，但确实起作用。研究中，83%可能拿到“鱼缸奖励”的病人坚持了12周治疗，普通疗法只有20%坚持下来。80%有“鱼缸奖励”的病人通过了药物测试，而普通疗法只有40%。治疗结束后，使用“鱼缸法”的人更不容易故态复萌——即便没有奖励承诺了。

这就证明了，难以预料的奖励究竟有多么强大的力量。和有保证的小奖励相比，我们的奖励系统面对可能获得的大奖会更兴奋。它促使我们去做任何可能获奖的事。这就是为什么人们宁愿买乐透彩票，也不愿意把钱存到银行里，赚取有保障的2%的利息。这也就是为什么，公司最底层的员工也相信自己有朝一日能成为CEO。

意志力实验：为了你的“我愿意”挑战分泌多巴胺。如果有什么事让你觉得很不愉快，所以你总是拖延不去做，你能不能把它和能让多巴胺神经元燃烧的事联系做一起，从而促使自己去做呢？

以下为作者的学生的办法

  • 带上可怕的文书工作走进自己喜欢的咖啡厅，边喝热巧克力边完成工作。
  • 买一堆刮刮乐彩票放着你想拖延的项目周围。
  • 想象自己努力工作后取得的最佳结果，让未来的奖励显得更加真实。

八、多巴胺的阴暗面

奖励的承诺带给我们的压力和快乐几乎不分上下。渴望并不是总让我们感觉良好，有时候它会让我们觉得自己堕落了。因为多巴胺的首要功能是让我们追求快乐，而不是让我们快乐。它并不在意给我们带来点压力，即便这会让我们在追求快乐的时候觉得不快乐。

深入剖析：欲望的压力。这一周，看你能否发现渴望会引发压力和焦虑。如果你屈服于诱惑的话，你觉得这是自己对奖励承诺的反应？还是在缓解焦虑？

我们误把渴望的感觉当作快乐的保证。奖励的承诺有很大的力量，它会让我们继续追求那些不会带给我们快乐的东西，会让我们消费那些不会带来满足感，只会带来更多痛苦的东西。追求奖励是多巴胺是主要目标，所以，即便你经历的事物和原本承诺的并不相符，它也不会给你释放“停下来”的信号。

对很多人来说，这个发现会让他们对奖励感到不满，然后远离奖励。吃薯片上瘾的人会带着怀疑的目光打量一包薯片。为了获得快乐而购物的人，她最喜欢去购物中心的感觉，但花钱让她有压力，于是信用卡放在家里，她从购物中心回家时，远比她花了很多钱时更快乐。

当你理解了所谓的“奖励”到底给自己什么感觉时，你就能做出最明智的决定，知道该怎么“奖励”自己了。

想一想你最大的“我不要”的意志力挑战吧。那很可能是一件让你觉得快乐的事——或者，如果你能得到足够多的话，它就会让你觉得快乐。但仔细分析一下那种经历和它的后果，你就会发现结果总是事与愿违。

如果强迫大脑，让它一边期待着奖励——获得快乐、幸福和满足，结束痛苦和压力，一边真切地感受现实状况，那么大脑最终会挑战它的期望值。

意志力实验：测试奖励的承诺。找一个常常让你放纵自己的诱惑因素测试一下。请关注你放纵的过程，不要急着去体验。注意这种奖励的承诺给你什么感觉。期待、希望、兴奋、焦虑、流口水……你的大脑和身体感觉到了什么。然后允许自己解释诱惑。和你的期望，这种体验怎么样？奖励的承诺有没有消失？它是否仍然促使你吃得更多、花得更多、待得更久？什么时候你会感到满足？你是否达到了一种没法继续的程度，因为你太饱了、太累了、太沮丧了、没时间了，或是无法得到“奖励”了？

进行这练习会有两种结果：

  1. 当他们真的关注放纵的感受时，他们实际上并不需要自己想象中那么多东西。
  2. 这种体验完全无法让他们满足。这就暴露了奖励的承诺和实际体验之间的差别。

这两种观察都会让你对曾经无法控制的事有更强的自控力。

九、欲望的重要性

当我们错把欲望当快乐时，我们就遇到麻烦了。这时候，解决问题的方法不是消除欲望。没有欲望的生活可能不需要这么多自控，但那也不能称为生活了。

大脑的“奖励系统”受损，快感缺乏的人认为生活就是一系列的习惯，他们没有对满足感的期待。他们可以吃东西、购物、社交，甚至有性生活，但不会期待从中获得快乐。当他们不再需要快感的时候，他们就失去了动力。如果你想不出任何一件让你感觉良好的事，你就很难从床上爬起来做事，这种毫无疑问的状态耗尽了希望，也夺走了很多人的生命。

当我们的奖励系统平静下来，我们并不会感到满足，而更可能表现得冷漠。这就是为什么很多帕金森病人会觉得抑郁，而不是安宁。神经科学家现在怀疑，不够活跃的奖励系统正是抑郁症的生理学基础。科学家观察抑郁症患者的大脑，发现即使唾手可得的奖励，这些人的奖励系统也不会变的活跃起来，系统并非完全不活动，只是不能创造出完整的“我想要”或“我想得到它”的感觉。

奖励的悖论

如果我们想拥有自控力，就需要区分让我们的生活有意义的真实奖励，和让我们分散精力、上瘾的虚假奖励。

学会区分这两种奖励，也许是我们能做到的最好的事了。这不并简单。

十、写在最后的话

欲望是大脑的行动战略。它可能对自控构成威胁，也可能是意志力的来源。当多巴胺让我们屈服于诱惑的时候，我们必须屈服渴望和快乐。我们也可以利用多巴胺和奖励的承诺来激励自己和他人。

欲望没有绝对的好坏之分，重要是欲望将我们引向那个方向，以及我们是否足够明智，知道什么时候该听从欲望的声音。

PE

2019-02-09

Win16平台上可执行文件是NE，在Win32平台上是PE（Portable Executable File Format）。EXE文件和DLL文件的区别是语义上的，它们使用相同的PE格式，仅用一个字段标识文件是EXE或DLL。

基本概念

PE文件使用一个平面地址空间，所有代码和数据合并一起，文件内容分割为不同区块，各个区块按页边界对齐。
PE文件不是作为单一内存映射文件被载入内存。Windows加载器遍历PE文件并决定文件哪一部分被映射，映射方式是将文件较高偏移位置映射到较高的内存地址中。载入内存后，磁盘和内存中的数据结构布局一致，但数据之间的相对位置可能会改变。

区块

1、区块表

PE文件头与原始数据之间的区块表，包含每个块在映像中的信息，分别指向不同区块。
区块表紧跟IMAGE_NT_HEADERS，是IMAGE_SECTION_HEADER数组。

2、区块合并

PE文件一般至少有代码块（.text）和数据块（.data）。
链接器的工作就是合并所有OBJ和库中需要的块，使其成为一个合适的区块。若使用/MERGE开关，链接时能合并两个有相似或一致属性的区块。优点：节省磁盘和内存空间。

3、区块对齐

  1. FileAlignment：磁盘区块对齐值，典型值200h，不足处被0填充，称为区块间隙。
  2. SectionAlignment：内存中区块对齐值。PE文件被映射到内存时，区块总至少从一个页边界处开始。x86系统一般为4KB（1000h），x86系统一般为8KB（2000h）。

4、输入表

1. 输入函数的调用

输入函数就是被程序调用但其执行代码不在程序中的函数，当PE文件载入内存后，Windows加载器才将相关DLL载入，并将调用输入函数的指令和函数实际所在地址关联。

PE文件中有一组对应被输入DLL的数据结构，每个结构都给出被输入DLL的名称并指向一组函数指针，这组函数指针称为输入地址表（Import Address Table，IAT）。

编译器无法区分输入函数调用和普通函数调用时，会出现低效调用：

    Call 00401164
    ......
:00401164:
    Jmp dword ptr[00402010]

对每个函数调用，编译器使用相同形式的CALL指令
CALL XXXXXXXX

“XXXXXXXX”是一个由链接器填充的实际地址，指令是从代码中的实际地址来的。JMP指令来自为输入函数准备的输入库。
优化方法：使用__declspec(dllimport)告诉编译器，函数来自另一个DLL，编译器会产生指令CALL DWORD PTR [XXXXXXXX]。
此外，编译器给函数加上__imp_前缀，将函数送给链接器，直接把__imp_xxx送到IAT，而不需要调用JMP stub。

2.输入表的结构

每个被PE文件隐式链接的 DLL 都有一个IMAGE_IMPORT_DESCRIPTOR（IID数组。

  1. OriginalFirstThunk：指向输入名称表（INT）的 RVA，INT 是IMAGE_THUNK_DATA结构的数组，数组中每个结构都指向IMAGE_IMPORT_BY_NAME结构。
  2. Name：DLL 名字的指针，以 “00” 结尾的 ASCII 字符的 RVA 地址。
  3. FirstThunk：指向输入地址表（IAT）的 RVA。

OriginalFirstThunk 和 FirstThunk 相似，指向两个本质相同的数组IMAGE_THUNK_DATA结构。
